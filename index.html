Resources Roles TA Lecture and Labs

Resources:

As discussed previously, this class has numerous components that will be utilized to present the material necessary for understanding the coursework, the project and collaboration aspects. The main source of communication of material will be SLACK (if account isn't made, do so immediately!). This is where one can reach out with the Faculty as well as Team Members, and can access Lecture Material and External Links. We will also use CANVAS as source where announcements will be made and assignments will be posted. Finally, we will use ZOOM in the instance that the class is made remote, however, this expectation is low since this class and its deliverables are expected to be IN-PERSON.

Roles:

The project is made up of 10 MEMBERS, where we will have TWO PROJECT LEADS and EIGHT DEVELOPERS. These developers will will also be split into two teams, FOUR who will focus on TIMELINE SCHEDULING and FOUR who will be PLANNING THE PROJECT. Ultimately, it is necessary for ALL MEMBERS TO CONTRIBUTE TO THE PROJECT, since this is where success is determined!

TA:

Each group is also assigned 1 TA, who is to act like the Project Manager for the group. The TA is to be the main source of information between the class and the project, and will act as the primary source of guidance for the project. The TA is meant to HELP YOU through this process, helping answer all queries, setting up tentative meetings and checking on the project's progress. PLEASE RESPECT THE TA's REQUESTS AND ADVICE, as doing so ensures the greatest potential to be drawn into the project.

Lecture and Labs:

As discussed already, it is imperative for members to be able to attend both LECTURES AND LABS. The lectures will be hosted by Professor Thomas Powell on Mondays, Wednesdays and Fridays from 5 to 6 pm at WLH 2001. The Labs for this class will be hosted on Thursdays from 9 am to 6 pm. Although attendance isn't mandatory, it is great practice to attend both Lectures and Labs to get the MOST of all material covered.


New Things Covered

Group Meetings Expectations Project Details Deadlines

Group Meetings:

It is expected for the project group to have WEEKLY MEETINGS with each other to discuss progress. These meetings are to be done with the TA, and we will conduct these weekly meetups every WEDNESDAY after the Lecture (i.e. past 6pm). The TA has also iterated to us that we should conduct DAILY DISCUSSIONS for around 30-45 minutes where the group can discuss ideas and results. These meetings should ideally be In-Person and not as much on Zoom, however, due to constraints, the first meeting was held online. EXPECT FURTHER MEETINGS TO BE IN PERSON

Expectations:

As described earlier, the project group consists of 11 members, 2 PROJECT LEADS, 8 DEVELOPERS and 1 TA. These 8 developers will be split into 2 groups as well, 4 FOR TIMELINE SCHEDULING and 4 FOR PLANNING. It is expected that project members devote at least 6 HOURS PER WEEK for the project. The project can only go so far if one person contributes, so it is imperative for the maximum participation of ALL MEMBERS. Questions should be raised and addressed immediately, progress is to be shared vividly, and collaboration is expected thoroughly. At the end of the day, it is an experience meant to help all of us! You will get in return whatever effort you put into this project!

Project Details:

In our first meeting, we discussed that the initial stage of the project is to familiarize ourselves with each other, understand the expectations of the project and depend on the element of TEAMWORK. In further meetings, we will discuss about the resources that will be utilized in the project and how it is to be delivered. We are expected to hold an ICEBREAKER event to help ease the group members to know each other, which is to be led by the project leads. The primary details of the project (what it is supposed to be) will be delivered to us on Friday by Professor Thomas Powell, and further supplements will be provided as the course progresses. The first half is to be light, but expect the second half to be devoted heavily to the project, with thorough implementation happening EVERY DAY. As mentioned earlier, it is necessary for COLLABORATION BY ALL MEMBERS for the most success of the project. We will review this project with our TA in our weekly meetings, and feedback will be given for the progress. This project will encompass the entire process of Software Development: Ideation, Specification, Development and Delivery. We will mimick the same pathway that is taken by Software Engineers in this industry. Again, you will get in return whatever effort you put into this project!

Deadlines:

The primary deadline for this project is set to be around FINALS WEEK (around first week of December). The project is to be completed by this timeframe. We will be presenting this project to the entire class, so be ready! Again, we are supposed to have weekly meetings with our TA on Wednesdays of each week until the Final Week, and it is expected we meet up for daily rundowns for 30-45 minutes. Potential deadlines for each part of the project are also to be provided by the TA and the Professor through the course to ensure the right progress is being conduct, so be alert for these. Remember, DO NOT WAIT TILL THE LAST MINUTE TO COMPLETE THE PROJECT.
